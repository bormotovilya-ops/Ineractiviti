# Шаблон интерактивного SCORM-сценария

Копия сценария **PeopleGames** («Корпоративные игры: Как не стать ПЕШКОЙ») со всеми файлами, необходимыми для запуска. Используйте эту папку как основу для нового проекта конструктора интерактивных сценариев на базе SCORM.

## Структура проекта

- **index.html** — одностраничное приложение курса (слайды, диалоги, квиз, сертификат). Точка входа.
- **imsmanifest.xml** — манифест SCORM (организация, ресурсы, точка входа).
- **js/** — скрипты: `scorm_api.js` (обёртка SCORM 1.2 для связи с LMS).
- **assets/images/** — все изображения: фоны (office.jpg, KITCHEN.png, OFFICE_NIGHT.png и др.), персонажи (boss..png, Леночка1/2, Виктор, Алиса и т.д.), Bern.png, cert.png, First.png и др.
- **assets/audio/** — аудио: mfon2.mp3, applause.mp3.
- **docs/** — документация: README.md, PeopleGames.md (описание структуры курса в формате Marp).

## Как запустить

1. **Локально (без LMS)**  
   Откройте `index.html` в браузере (через файловый протокол или любой статический сервер). Курс работает автономно; SCORM-функции не будут вызываться, пока нет LMS.

2. **В LMS**  
   Упакуйте содержимое папки в ZIP и загрузите как SCORM 1.2-пакет. Точка входа — `index.html`.

3. **Через dev-сервер проекта MYMiniapp**  
   Если папку положить в `public/Courses/` (например, как подпапку курса), курс будет доступен по пути вида `/Courses/ScormScenarioTemplate/index.html`.

## Зависимости из основного проекта

- **Аналитика в LMS (Интерактивити):** при запуске через `landing/scorm-player.html` плеер подставляет `window.__PEOPLE_GAMES_COURSE_ID__` и пишет в общие таблицы LMS (`course_analytics_*`, RPC `course_analytics_record_run` / `course_analytics_record_choice_event`). Сводка в кабинете — `course_analytics_get`. SQL: `tech/supabase-course-analytics-lms.sql` (одна схема на все курсы).
- Локальный офлайн-режим по-прежнему копит черновую статистику в `localStorage` (`peoplegames_run_stats`), без записи в БД.

Внешние ресурсы: шрифты Google Fonts, Tailwind и GSAP с CDN — курс работает при наличии интернета.

## Дальнейшие шаги

- Переименуйте папку под свой сценарий.
- Замените контент (тексты, изображения, аудио), сохраняя структуру слайдов и вызовы `js/scorm_api.js`.
- При необходимости обновите `imsmanifest.xml` (identifier, title, список `<file>` в `<resource>`). Все пути к файлам должны соответствовать структуре папок (js/, assets/images/, assets/audio/).
- Добавьте свой API аналитики/кабинета или уберите соответствующие блоки в `index.html`.
